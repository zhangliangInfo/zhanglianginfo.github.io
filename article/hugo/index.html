用Hugo搭建blog

Wed, Aug 12, 2020

今天使用hugo搭建了一套blog系统，用来代替记事本功能，来记录一下琐碎的工作学习的过程：

1.关于项目安装配置

由于我的是windows系统，先说下windows下的配置安装：

  • 首先是下载hugo对应的版本，传送门
  • 创建工作目录

$:cd D:
$ mkdir hugo
$ cd hugo $ mkdir bin

  • 将hugo_0.74.3_Windows-64bit.zip下载文件里的hugo.exe文件复制到bin目录下
  • 配置环境变量：D:/hugo/bin/hugo.exe

$:hugo new mysite

  • hugo会自动生成项目,目录结构如下：

▸ archetypes/
▸ content/
▸ layouts/
▸ static/
config.toml

  • 进入到项目cd mysite
  • 通过hugo new xxx.md可以在content目录内创建一个md文件，语法为markdown语法

$ cd thems
$ git clone https://github.com/spf13/hyde.git
$ hugo server –theme=hyde –buildDrafts启动项目

  • 浏览器打开http://localhost:1313可以查看效果

2.如何部署到github Pages

  • 首先在github上创建一个Repository： example.github.io
  • 在站点根目录设置项目链接

$ hugo –theme=hyde –baseUrl="http://example.github.io”

  • 所有静态页都会生成到public目录下，将public目录下所有文件提交到创建的github仓库中即可
  • 执行hugo命令打包html到public

$ hugo -t hyde

若发现没有生成html，将md文件头部的draft=true删除或改为false, 或者在上述命令后加 –buildDrafts | -D(将生成被标记为草稿的页面)

$ cd public
$ git init
$ git remote add origin https://github.com/example/example/github.io.git
$ git add –all $ git commit -m ‘initial commit’
$ git push -u origin master(若提交失败，可执行git push -f origin master)

  • 推送成功后，那么恭喜你，可以直接访问自己的blog：http://example.github.io
